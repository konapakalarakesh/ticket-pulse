How this dashboard is wired

Three moving parts, all static — nothing runs on a server you have to maintain.

  1. The audit sheet — this repo has just two files: index.html and the audit workbook sitting right next to it. Whenever the sheet changes, upload the new version to GitHub with the exact same file name so it overwrites the old one — no code edit needed. The dashboard fetches it same-origin from the published Pages site, so there's no CORS issue and no separate hosting step.
  2. Live ticket status — t.corp.amazon.com doesn't expose a public API, and a page on github.io can't read t.corp's DOM or cookies across origins. So instead of faking it, this dashboard opens the real ticket page in a small popup (using your authenticated session) and relies on a one-time browser userscript to read the status off the page and hand it back. This is the only reliable way to get true live data into a GitHub-hosted page.
  3. Everything else — theme, cache, chart state — lives in your browser's local storage. Nothing is sent off your machine.

1. Install the helper userscript (one-time, ~2 minutes)

  1. Install the Tampermonkey browser extension (already allowed on most corp images).
  2. Click ⬇ Download helper script below — it saves ticket-pulse-helper.user.js straight from this page (it's built into the dashboard, not a separate upload). Open Tampermonkey → Create a new script → delete the boilerplate → paste in the downloaded file's contents → save.
  3. Edit the TARGET_ORIGIN constant near the top of that file to match your dashboard's exact URL (e.g. https://your-org.github.io) — this stops any other site from receiving your ticket data.
  4. Visit any ticket at t.corp.amazon.com/<id> once while signed in — you should see a small "✓ Synced to Ticket Pulse" toast appear. That confirms it's working.

2. Point the dashboard at your repo

Open index.html and edit the CONFIG block at the top of the script:

const CONFIG = {
  AUDIT_DATA_URL: './Audits.xlsx',  // same-origin file next to index.html, cache-busted automatically
  TICKET_BASE_URL: 'https://t.corp.amazon.com/',
  STATUS_CACHE_TTL_HOURS: 6,  // how long a checked status is trusted before it's "stale" again
  AUTO_REFRESH_MINUTES: 15
};

3. Publish

Push this folder to a repo, enable GitHub Pages under Settings → Pages (branch: main, folder: root), and share the resulting https://<org>.github.io/<repo>/ link with the team. Anyone on the corp network with Midway access can use it as-is.

Good to know

  • Without the userscript installed, the checker still works in manual mode — click "Open ticket ↗" and set the status yourself from the dropdown. Manual entries are tagged so the team knows they weren't auto-verified.
  • Popups need to be allowed for this site the first time you run a live check — your browser will prompt once.
  • Status is cached per browser for 6 hours so a refresh doesn't re-open every ticket; just re-run the checker on a ticket to force an update sooner.
